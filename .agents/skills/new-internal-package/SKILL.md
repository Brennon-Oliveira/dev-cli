# New Internal Package

Use this workflow when scaffolding `internal/{pkg}`.

## Shape The Package

1. Read `internal/AGENTS.md` and nearby packages with similar responsibility.
2. Create the narrowest package needed under `internal/{pkg}`.
3. Prefer the established file pattern:
   - `{pkg}.go` for interfaces and public package types.
   - `{pkg}_builder.go` for `New...` and option functions.
   - `{pkg}_impl.go` for the concrete implementation.
   - `{pkg}_test.go` for focused unit tests.
   - `{pkg}_mocks.go` only when generated by Mockery.
4. Use existing interfaces from `internal/exec`, `internal/logger`, `internal/config`, or other packages instead of shelling out directly when a local abstraction exists.

## Interfaces And Mocks

If the package defines an interface that tests or commands will mock:

1. Add the package to `.mockery.yml` using the existing package entries as the template.
2. Run `mockery --all` if Mockery is available.
3. If Mockery is not available, state that mock generation was not run and leave clear follow-up instructions.

## Testing

Run targeted tests first:

```bash
go test -v ./internal/{pkg}
```

Then run:

```bash
go test -v ./...
go build -o dev main.go
```

Keep tests isolated and use testify assertions, matching existing package tests.
